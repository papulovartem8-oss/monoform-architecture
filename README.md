# MONOFORM — Architecture & Interiors

Премиальный одностраничный сайт архитектурного бюро на Next.js App Router, TypeScript и Vinext. В проекте есть адаптивная редакционная композиция, GSAP ScrollTrigger, Lenis, интерактивный кейс, форма-заглушка и подготовка к публикации через Sites.

> Все изображения в репозитории — временные демонстрационные материалы, сгенерированные специально для макета. Они не являются реальными работами MONOFORM и должны быть заменены перед коммерческой публикацией.

## Запуск

Требуется Node.js 22.13 или новее.

```bash
npm install
npm run dev
```

Для проверки production-сборки:

```bash
npm run lint
npm run build
npm test
```

## Быстрая персонализация

Основной контент собран в `src/content/site.ts`. В одном месте можно заменить:

- название, подпись, географию и годы работы;
- навигацию и тексты всех секций;
- проекты, метаданные и alt-тексты;
- этапы кейса, помещения и материалы;
- услуги, процесс и контакты;
- базовые цвета и акцент.

Логотип сейчас текстовый. Чтобы использовать графический знак, добавьте файл в `public/brand`, укажите путь в `site.ts` и замените текстовый `.wordmark` в `app/components/MonoformHome.tsx` на `next/image`.

## Замена проектов и изображений

Каждый проект хранится в собственной папке:

```text
public/projects/
  house-on-the-ridge/hero.webp
  silent-apartment/hero.webp
  forest-residence/hero.webp
  material-gallery/hero.webp
```

Рекомендуемые исходники:

| Назначение | Пропорция | Рекомендуемый размер |
| --- | --- | --- |
| Hero и полноэкранный проект | 16:9 | 2560×1440 px или больше |
| Вертикальный проект | 3:4 | 1800×2400 px или больше |
| Горизонтальный проект | 2:1 | 2800×1400 px или больше |
| Изображение кейса | 3:2 | 2400×1600 px или больше |
| Open Graph | 1.91:1 | 1200×630 px |

Используйте WebP или AVIF с качеством 80–88. После добавления файлов обновите `image` и `alt` у проекта в `src/content/site.ts`. Компонент `next/image` резервирует место и лениво загружает все кадры за пределами первого экрана.

## Подключение формы

Сейчас форма показывает локальное состояние успеха и ничего не отправляет. Точка интеграции находится в функции `submitForm` файла `app/components/MonoformHome.tsx`.

Замените локальный `setSent(true)` на `fetch` к вашему API route, CRM или Telegram-боту. Рекомендуется:

1. создать серверный endpoint;
2. валидировать и очищать данные на сервере;
3. добавить защиту от спама и rate limit;
4. показывать отдельные состояния загрузки и ошибки;
5. не хранить ключи и токены в клиентском коде.

## Перед публикацией

- замените все временные изображения реальными материалами бюро;
- проверьте email, телефон и Telegram в `site.ts`;
- проверьте корректность canonical-адресов на рабочем домене (metadata, robots и sitemap автоматически используют домен запроса);
- обновите `public/og.png` и favicon при смене бренда;
- подключите форму к backend и добавьте юридические ссылки;
- проверьте права на фотографии и шрифты;
- выполните финальный прогон на реальных мобильных устройствах.

## Motion и доступность

На desktop используются mask reveal, controlled image scale, ScrollTrigger-сцены, плавный скролл Lenis, кастомный курсор и магнитный CTA. На мобильных pinned-блоки упрощены, custom cursor отключён, а интерактивы доступны через tap и клавиатуру. Для `prefers-reduced-motion` тяжёлые эффекты отключаются.
